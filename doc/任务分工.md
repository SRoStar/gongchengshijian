# 在线化学知识库（前后端分离 + Agent）四人分工方案

整体按**前端、后端、AI Agent、测试 / 运维 + 文档**四大角色划分，职责边界清晰、协作闭环，适配中小型团队开发，兼顾功能迭代与上线落地。

## 一、人员整体定位

4 人分工：**前端开发 1 人、后端开发 1 人、AI Agent 开发 1 人、全栈测试 / 运维 / 产品 1 人**，覆盖从界面、接口、智能能力到上线、维护全流程。

------

## 二、各角色详细职责

### 角色 1：前端开发（成员 A）

**核心负责**：网站页面、交互、前后端联调、适配

1. 搭建项目框架：Vue/React + Vite，实现**前后端分离架构**
2. 页面开发：知识库首页、化学分类目录、知识点详情、搜索页、个人中心、Admin 后台页面
3. 交互功能：文档浏览、目录跳转、关键词检索、收藏 / 点赞、分页、富文本化学公式渲染
4. 对接接口：联调后端知识库 CRUD、搜索、分类接口；对接 AI Agent 对话弹窗、问答流式输出
5. 样式优化：响应式布局、化学公式 / 结构式展示、动画交互、兼容性适配

### 角色 2：后端开发（成员 B）

**核心负责**：业务接口、数据库、权限、基础服务

1. 技术选型：Java/SpringBoot/ Python/Django + MySQL/Elasticsearch（检索优化）
2. 数据库设计：化学知识点表、分类表、标签表、用户表、收藏表、日志表
3. 业务接口：知识点增删改查、分类管理、全文检索、用户登录 / 权限、文件上传（化学资料）
4. 中间件 & 部署支撑：接口鉴权、数据缓存、分页、数据导入导出
5. 对接 AI Agent：提供**统一转发接口**，承接前端请求、调用 Agent 服务、返回结果，做异常拦截

### 角色 3：AI Agent 开发（成员 C）

**核心负责**：智能问答 Agent、知识库语义理解、大模型对接（项目核心特色）

1. Agent 架构搭建：对话流程设计、上下文管理、会话记忆
2. 知识库联动：**检索增强生成 (RAG)** 开发 —— 向量库搭建、化学文档切片、向量化、相似度召回
3. 大模型对接：接入大模型 API / 本地模型，编写 Prompt（化学专业问答限定、公式解析、知识点答疑）
4. 能力开发：化学知识点解答、公式推导、错题解析、相关知识推荐
5. 接口封装：独立 Agent 服务接口，供后端统一调用，处理流式问答、超时、重试逻辑
6. 专业优化：针对化学术语、结构式、反应方程式做语义适配

### 角色 4：测试 + 运维 + 产品 + 文档（成员 D，统筹兜底）

**核心负责**：需求梳理、测试、部署、运维、项目文档、协作协调

1. 产品 & 需求：梳理化学知识库功能清单、原型评审、需求变更管控
2. 测试工作：功能测试、接口测试、联调测试、Agent 问答效果测试、兼容性 / 压力测试，输出 bug 清单
3. 运维部署：服务器配置、前后端 + Agent 服务打包上线、域名 / HTTPS、容器部署（Docker）、日志监控
4. 数据维护：初始化化学知识库基础数据、数据校对、后台内容录入规范制定
5. 项目文档：接口文档、部署手册、使用手册、项目总结，协调四人跨模块沟通

------

## 三、模块协作流程（四人配合逻辑）

1. **前期**：D 出原型 + 需求 → 全员评审，确定技术栈与接口规范
2. 并行开发
   - A 写页面 + 模拟接口调试
   - B 建库 + 开发基础业务接口
   - C 搭建向量库 + RAG + Agent 对话能力
3. 联调阶段
   - A ↔ B：联调知识库常规功能
   - B ↔ C：联调 Agent 服务调用、数据流转
   - A ↔ C（间接）：通过后端接口完成前端问答交互
4. **测试上线**：D 全量测试 → 修复 bug → D 统一部署上线 + 线上运维

------

## 五、技术栈参考（配套分工）

- 前端：Vue3 / React + Element Plus / Ant Design
- 后端：SpringBoot + MySQL + Elasticsearch（检索）/或者用python，都可以
- AI Agent：Python + LangChain/LLamaIndex + 向量数据库（FAISS/Chroma/Milvus）+ 大模型 API
- 部署：Docker + Nginx，统一由运维负责人维护
